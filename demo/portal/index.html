Problem Statement

Every serverless function in India that touches personal or payment data is one misconfigured IAM policy, one missing consent check, or one non-India API call away from a DPDP violation - and no existing FaaS platform enforces these rules automatically at runtime.

VICT is a sovereign serverless security layer that intercepts every function invocation before execution, enforces citizen consent in real-time, blocks data from leaving India at the kernel level using eBPF, and generates a tamper-proof forensic trail - turning DPDP compliance from a quarterly audit into a continuous, automated guarantee.

Shield Real-Time Consent (DPDP)

Intercepts invocations to verify Data Principal consent via a secure Identity Bridge.

Map Kernel-Level Geofencing (RBI)

Uses eBPF to block non-India API calls at the OS level, ensuring data never leaves the territory.

Gavel Automated Forensics

Generates audit-ready logs that meet the 72-hour breach reporting requirement of the DPDP Act.
